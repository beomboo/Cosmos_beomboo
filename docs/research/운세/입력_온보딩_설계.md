# 입력 정보·온보딩 설계

_최종 확인: 2026-07-13_

## 필수 입력값
| 항목 | 비고 |
|---|---|
| 생년월일 | 양력/음력 선택 토글 필요 |
| 태어난 시간 | 24시간 기준, 분까지 받으면 정확도 상승 |
| 성별 | 대운(순행/역행) 계산에 필요 |

## 선택/고급 입력값
| 항목 | 비고 |
|---|---|
| 출생지(도시) | 진태양시 보정(경도)에 사용 — 없으면 표준시로 근사 |
| 윤달 여부 | 음력 입력 시 윤달 표시 필요 |

## "태어난 시간을 모를 때" 처리
- **엄격 요구형**: 시간 입력을 필수로 강제하되 "모름" 옵션은 두고, 시주 없이 근사 결과 제공(신운세 등)
- **유연 허용형**: 시간 없이도 진행 가능, "시주가 빠진 사주는 해석의 일부만 가능"이라고 안내
- **권장안**: "모름" 선택 시 시주만 제외한 3주(년/월/일)를 보여주고 "태어난 시간을 알면 더 정확한 결과를 볼 수 있어요" 넛지 문구로 재입력 유도

## 자시(子時) 관련 UX 안내
23:00~01:00 출생자는 일주가 전날/당일 중 어느 쪽으로 계산되는지 앱마다 다르다 — 이 구간 입력 시 "정확한 결과를 위해 병원 기록상 시간을 확인해보세요" 툴팁 고려. 실제 계산 관법 3가지(`midnight`/`splitJasi`/`jasi`)와 우리 코드의 현재 선택은 [../사주팔자/계산_정확도_이슈.md](../사주팔자/계산_정확도_이슈.md) 참고.

## 온보딩 흐름 (경쟁 앱 공통 패턴)
1. 생년월일시 입력(달력 UI+시간 스크롤 피커)
2. 양력/음력 토글
3. 성별 선택
4. (선택) 출생지 입력 — 스킵 가능하게
5. 로딩 애니메이션(계산 중 — 캐릭터/일러스트 연출 넣기 좋은 지점)
6. 결과 화면 진입

## UX 주의점
- 입력 필드가 많으면 이탈률 상승 → 필수값 최소화+선택값은 "나중에 추가" 유도
- 시간 모름 케이스가 실제로 매우 흔함(병원 기록 없는 경우 다수) → 이 케이스의 결과 품질/설명이 앱 신뢰도에 직결
