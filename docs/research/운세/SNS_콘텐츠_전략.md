## 포스텔러 콘텐츠 유형 4가지 (최근 12건 분석)
1. **엔터테인먼트/퀴즈형**: "남자 얼굴로 보는 나의 사주 취향" 등 — 좋아요 700~1,000+로 가장 반응 좋음
2. **일간(일주) 밈화**: "무자일주가 단톡에서 읽씹하는 이유ㄷㄷ" — 명리학 용어를 밈 문구로 캐릭터화
3. **아이돌/연예인 콜라보**: 팬덤 트래픽 유입, 유튜브 풀버전 연계
4. **일상 운세 CTA형**: "댓글로 '행복해져라' 외쳐보세요" — 참여 유도 고정 멘트로 인게이지먼트 확보

## 신살 밈 카피 포맷 — "체크리스트형 자가진단"
포스텔러의 "인간 도화살이 있다?"도 이 신살 카테고리였음이 확인됨. Threads(@mystic_bluewater) 실제 게시물:
> 〈사주에 도화살(桃花殺) 있는 사람들 특징〉 1~7번 일상 경험 체크리스트 + **"3개 이상이면 100% 보유자"**

기존 "개념 하나→유행어 하나" 밈화와 다른 구조 — **번호 체크리스트+정량적 자가진단 임계값**. 자세한 내용은 [../사주팔자/신살.md](../사주팔자/신살.md).

## 점신 vs 포스텔러 — 정반대 전략
같은 사주 카테고리 안에서도 전략이 완전히 다르다:
- 점신: 해시태그 SEO 스터핑(12지지/별자리 전부 나열), 요일별 루틴형 카드뉴스, 참여 유도 장치 없음(댓글 대부분 0~7건)
- 포스텔러: 밈/퀴즈/아이돌 콜라보(엔터테인먼트), 참여 유도 멘트로 활발한 댓글

**다운로드·MAU는 점신이 앞서지만(1,700만 vs 750만), 인스타그램 인게이지먼트는 포스텔러가 압도적 우위** — 두 앱의 성장 채널 자체가 다르다는 뜻(점신은 광고·검색, 포스텔러는 SNS 바이럴).

## 성장 채널 3유형 종합
캐주얼 콘텐츠형(포스텔러·점신)과 상담 중개형(천명)은 SNS 마케팅 의존도 자체가 다르다. 우리 앱은 캐주얼 콘텐츠형에 가까우므로 포스텔러·점신 사례가 더 직접 참고 대상.

## 우리 앱에 주는 시사점
"명리학 개념을 밈화하는 것이 반응이 좋다"는 결론이 포스텔러·점신 직접 대비로 더 뚜렷해짐 — 우리는 이미 캐주얼 톤을 목표로 하므로, SNS 마케팅을 한다면 점신식 "요일별 정기 운세" 반복보다 **포스텔러식 "밈/퀴즈/유형 진단" 콘텐츠**가 더 유리.

## SNS 채널 재시도 결과 (2026-07-15, 2026-07-17 재확인) — Facebook/B站/샤오홍슈/V2EX

기존 README "Agent-Reach 채널 현황" 표에서 "미검증"으로 남아 있던 4개 채널을 실제로 재시도한 결과. 2026-07-17 오버나이트 루프에서 동일 채널을 한 번 더 재시도했다.

### Facebook · B站(bilibili) · 샤오홍슈(小红书) — 재시도 실패 재확인(2026-07-17)
`opencli doctor` 진단 결과 이번에도 `[FAIL] Connectivity: Browser Bridge extension not connected`(Chrome 확장 미연결)로 동일했다. `opencli facebook whoami`·`opencli bilibili whoami`·`opencli xiaohongshu whoami` 세 명령 모두 15초 내 응답이 없어 강제 종료했다(무인 세션 특성상 `timeout`/`gtimeout` 바이너리가 없어 백그라운드 실행 후 수동 kill로 확인). 세 채널 모두 로그인 세션(`[cookie]`)이 필요한 OpenCLI 브라우저 브리지 채널이라, 브라우저 확장이 상시 연결된 사람 세션이 아니면 원천적으로 호출 자체가 불가능하다는 기존 결론이 이번 세션에서도 그대로 재확인됐다 — 새로운 정보는 없음.

### V2EX — 공개 API 성공, 단 제한적으로 유용
`opencli v2ex hot`/`latest`/`nodes`/`topic <id>`/`replies <id>`는 브라우저 세션 없이 정상 동작한다(✅). 다만 **풀텍스트 검색 명령이 없어**(`hot`/`latest`/`nodes`로 목록만 가져올 수 있고 키워드 검색은 불가) 실전에서는 `site:v2ex.com <키워드>` 형태의 Exa 웹 검색으로 관련 글타래를 먼저 찾은 뒤 `opencli v2ex topic <id>`로 본문·댓글을 조회하는 하이브리드 방식이 필요했다. 2026-07-17에도 `hot`/`latest` 목록 자체에는 사주 관련 글이 없어 여전히 이 하이브리드 방식으로 새 글을 찾았다.

V2EX는 중국어 프로그래머/인디 개발자 포럼이라 "SNS 마케팅 레퍼런스"보다는 **"제품·개발자 커뮤니티 반응" 자료**에 가깝다. 2026-07 상순부터 사주(八字/BaZi) 관련 인디 개발자 프로젝트가 짧은 기간에 몰려서 올라와 있어, 교차 시장(중국) 검증 자료로 기록해둘 가치가 있다.
