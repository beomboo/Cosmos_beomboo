# UI/UX 트렌드

_최종 확인: 2026-07-17_

> 결과 화면의 구체적 레이아웃·카테고리 카드·공유 카드 사례는 [결과화면_설계.md](결과화면_설계.md)에 정리되어 있다 — 이 문서와 역할이 겹치지 않도록, 여기서는 "전반적 UI/UX 방향성·컬러·톤" 수준만 다룬다.

## 핵심 방향
"무거운 점술" → "가벼운 자기이해 콘텐츠". 전통 사주 상담소의 무거운 분위기를 벗고 모바일 최적화된 직관적·재미있는 경험으로 재해석하는 것이 MZ세대 공략의 핵심.

## 1. 캐릭터화
MZ세대는 캐릭터화된 운세를 선호 — MBTI처럼 "나는 어떤 동물/기운 타입일까?" 라벨링 게임 형식이 효과적. 최고심 브랜드의 스크래치 복권형 부적 카드("돈벼락"·"합격" 문구+귀여운 일러스트), 카카오·토스도 자체 캐릭터로 디지털 부적 제작. 캐릭터 기반 UX가 실제로 고평점(별점 4.8, 리뷰 1.7만+)을 받은 실측 사례는 "우주고양이 보라"([결과화면_설계.md](결과화면_설계.md) 참고) — 캐릭터화가 이론이 아니라 시장에서 검증된 방향임을 뒷받침.

## 2. 컬러 팔레트
밝은 원색보다 **소프트 톤(파스텔) 선호도 높음**. Z세대 팔레트: 피부/베이스=피치·소프트 핑크, 포인트=라벤더·민트 그린, 서브=세이지 그린·페리윙클, 강조(CTA)=코랄·터쿼이즈.

**2026-07-17 재조사 시 발견한 반대 신호 (일반 웹디자인 트렌드 예측)**: Adobe·Canva·WIX 등이 발표한 "2026년 UI/UX 디자인 트렌드" 자료들은 공통적으로 "뉴트럴 톤·파스텔"에서 "눈에 띄게 밝고 채도 높은 색"으로 이동한다고 전망한다(WIX 크리에이티브 디렉터 발언 인용). 다만 이 예측은 **웹사이트·브랜드 캠페인 전반**을 다루는 일반론이며, 국내 사주/운세 버티컬 앱을 특정한 근거는 아니다 — 실제로 확인된 사주 앱들(포스텔러·헬로우봇·우주고양이 보라, GitHub 소스까지 직접 확인한 Dr.Saju는 오히려 반대로 다크+골드 프리미엄 톤)은 여전히 파스텔 또는 극단적 톤(다크) 둘 중 하나로 갈릴 뿐, "채도 높은 원색"을 쓰는 사례는 확인되지 않았다. 따라서 우리 앱의 파스텔 큐트 채택 근거는 아직 유효하다고 판단하되, 차기 리서치 사이클에서 실제 사주 앱 신규 출시 사례 중 고채도 팔레트를 쓰는 곳이 나타나는지는 계속 관찰할 필요가 있다.

## 3. 언어/카피 톤
명리학 전문 용어(십신, 편관 등) 그대로 노출 금지 — 일상어로 재해석. "내 2026년 키워드는 OO" 같은 한 줄 요약형 카피가 SNS 공유에 효과적.

## 4. SNS 바이럴 설계
결과 화면을 인스타그램 스토리 규격(9:16)에 최적화. 공유 버튼은 하단에 묻히지 않게 상단/CTA 위치에 배치. 구체적인 공유 카드 디자인 규격·채널별 고려사항(카카오톡 정사각/가로 썸네일 등)은 [결과화면_설계.md](결과화면_설계.md)의 "SNS 공유용 결과 카드 디자인" 절에 정리.

## 5. 인터랙션 요소
로딩(계산 중) 구간에 캐릭터 애니메이션 배치(대기 시간의 콘텐츠화) — 이 지점은 [입력_온보딩_설계.md](입력_온보딩_설계.md)의 온보딩 흐름 5단계("로딩 애니메이션")와도 맞닿아 있다. 그 밖의 결과 전달 인터랙션(대화형 챗봇·스크래치/카드 뒤집기 게임화 등) 5가지 유형은 [결과화면_설계.md](결과화면_설계.md)의 "결과 전달 방식 5유형"에 통합 정리했으므로 여기서는 중복 서술하지 않는다.

## 6. 온보딩 vs 스토어 스크린샷
스토어 스크린샷은 "설치 유도", 온보딩 화면은 "서비스 이해·기능 활용" — 서로 다른 메시지로 설계. 온보딩 필수/선택 입력값과 흐름 자체는 [입력_온보딩_설계.md](입력_온보딩_설계.md)에 정리.

## 디자인 방향성 두 갈래 — 실제 소스 코드로 확인된 대비

| | **파스텔 큐트형**(포스텔러·헬로우봇) | **다크 프리미엄 미스틱형**(Dr.Saju 실제 코드 기준) |
|---|---|---|
| 배경 | 흰색/밝은 배경 | `#0a0a0f` 거의 검정 |
| 포인트 컬러 | 파스텔(피치·라벤더·민트) | 보라 `#8b5cf6` + 골드 `#C5A44E` |
| 톤 | 귀엽고 가벼움, 밈/게임화 | 신비롭고 고급스러움 |
| 캐릭터 | 귀여운 마스코트 중심 | 캐릭터별 챗 라우트는 있지만 비주얼은 절제된 다크 UI |
| 타겟 인상 | 10대~20대 초반 | 20대 후반~30대, "믿을 만한 프리미엄" 인상 |

**우리 앱은 파스텔 큐트를 채택**(`docs/mockups/01-pastel-cute.html`).

폰트 DM Sans, 모노스페이스 Geist Mono. shadcn/ui + daisyUI, radius `0.75rem` 기본. 마이크로 인터랙션: `glowPulse`(보라 글로우 펄스)·`shimmer`·`marquee`·`fadeIn/slideUp/scaleIn` 스태거 애니메이션 다수 — 정적 리포트가 아니라 등장 연출에 공들인 구조.

### 실제 사용자 플로우 (`src/components/saju/` 폴더명 기준)
```
landing → input → preview → result → report
                                 ↓
                    share / referral / upsell → payment → coin-shop
chat/[characterId] → (캐릭터별 AI 채팅 결과 해석)
```
- **result → report**: 요약형 결과 → 상세 리포트(유료)로 확장되는 지점이 명확히 분리 — 무료/유료 경계 설계에 참고([결과화면_설계.md](결과화면_설계.md)).
- **share / referral**: 결과 공유·추천인 초대가 결과 화면 바로 다음 단계 — 바이럴 루프가 플로우에 내장됨.
- **upsell → payment → coin-shop**: 코인(재화) 구매 방식 과금, 리포트 1건당 코인 차감 구조로 추정.

## 참고 경쟁 앱 UI 포인트
- **포스텔러**: 캐릭터 기반, 무료 콘텐츠 진입장벽 최소화. 리뷰 "캐릭터들도 진짜 귀엽고 사랑스럽다"
- **점신**: '운세보고서' 형식(오늘의 운세·총운·시간대별·바이오리듬·추천소식 종합), 상담사 랭킹으로 신뢰 부여
- **헬로우봇**: 흰 배경+굵은 글씨+둥근 카드, 캐릭터 '라마마'가 챗봇 형태로 대화하듯 결과 전달
- **Dr.Saju**: 8가지 캐릭터와의 AI 챗 형태(`chat/[characterId]`)로 결과 전달

다른 신규/리뉴얼 경쟁 앱(사주한눈에·우주고양이 보라·정통사주·2026 사주 카드)의 UI 포인트는 오행 시각화·카테고리 카드 구성과 함께 [결과화면_설계.md](결과화면_설계.md)에 정리되어 있어 여기서는 중복 나열하지 않는다.
