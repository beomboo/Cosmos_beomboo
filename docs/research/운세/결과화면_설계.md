# 결과 화면 설계 사례

_최종 확인: 2026-07-16_

## 결과 전달 방식 5유형
1. **정적 리포트형**: 명식+오행 그래프+텍스트 풀이(전통 만세력 사이트, 정확하지만 딱딱함)
2. **한 줄 요약+확장형**: "당신의 2026년 키워드는 OO" 헤드라인 → 탭하면 상세 풀이(포스텔러류, SNS 공유 최적)
3. **대화형(챗봇)**: 캐릭터가 대화하듯 순차 설명(Dr.Saju `chat` 라우트, 8종 캐릭터)
4. **카드/게임화형**: 스크래치·카드 뒤집기로 결과를 하나씩 공개
5. **계산은 앱, 해석은 외부 AI 위임형** — 아래 상세

### 5번 상세: "만세력 천을귀인"(안드로이드 만세력 앱 1위, 누적 40만)
앱은 사주팔자·대운·세운·십신·신살 등 정밀 데이터만 계산하고, "AI 사주풀이 프롬프트 생성" 기능으로 그 데이터를 담은 프롬프트를 자동 생성 — 사용자가 복사해 ChatGPT/Gemini에 붙여넣어 해석받는 구조. 앱은 AI API 비용·연동 책임을 전혀 안 지면서 "AI 해석"이라는 가치를 제공하는 하이브리드 방식 — 우리 앱의 AI 해석 연동이 사람 결정 대기인 상황에서 **저비용 대안**으로 고려할 만함(트레이드오프: 사용자가 직접 복사해야 하는 번거로움).

**실제 프롬프트 원문**(네이버 블로그 실사용 후기):
> "저의 사주를 심층적으로 해석해 주세요. 태어난 날짜는 [년/월/일/오전 or 오후 00시 00분]이며, [경진년 정해월 경진일 갑오시]간지의 성별은 [여 or 남]입니다. 저의 성향, 강점, 약점, 타고난 기질, 성격적 특징을 상세히 분석해 주세요."

필요한 최소 정보는 ①생년월일시(양력) ②4기둥 60갑자 명식(한글, "년주 월주 일주 시주" 순) ③성별, 세 가지뿐. **반드시 이미지가 아니라 이미 계산된 4기둥 텍스트를 프롬프트에 넣어야 한다**(이미지로 캡처해 보내면 AI가 대운 방향·간지 순서를 제대로 못 읽는 실사용 문제 확인됨) — 계산은 앱이 맡고 AI에는 결과값만 텍스트로 넘겨야 신뢰 가능.

**우리 프로젝트 시사점**: 이 프롬프트 데이터(4기둥 명식·생년월일시·성별)는 `share_text.dart`의 `buildShareText`가 이미 거의 동일하게 조합 중 — 기존 공유 텍스트 조립 로직을 살짝 변형해 "AI에게 물어보기"(프롬프트를 클립보드에 복사) 버튼 하나만 추가하면 저비용 프로토타입 가능.

- 성격/기질 분석

- 궁합 보기로 연결되는 CTA

- "카드 한 장 + 공유"만으로 앱 정체성을 세운 사례(2026 사주 카드)가 실존한다는 사실은, 오늘 강화한 공유 카드 기능의 방향이 시장에서 이미 검증된 패턴임을 뒷받침한다.

## 무료/유료 경계 — 3강 앱은 서로 다른 모델을 쓴다

### 1. 포스텔러 — 캐시(포스) 기반 콘텐츠 단위 결제
500포스(약 5,000원) 충전, 첫 결제 1+1 이벤트. 궁합 프리미엄 22,000원 사례 확인. 무료(짧은 요약)/유료(개인화 심층 풀이) 차이가 분명 — "운세 콘텐츠 단가가 좀 있는 편" 반복 언급. 콘텐츠 볼륨 대비 기대 괴리 후기도 존재("AI 상담 캐릭터가 생각보다 짧았다") — 유료 결제 후 기대 대비 짧으면 재구매 의사가 낮아질 수 있음을 시사.

### 2. 점신 — 광고 기반 완전 무료
개발자 직접 답변: "무료와 유료버전은 광고의 노출 유무 외에는 현재 동일한 기능". 대신 광고 강도 불만이 많음(30초+15초 연속 광고 등). **시사점**: 광고 기반은 무료 이용자 규모(1,800만+) 극대화엔 좋지만, "결제해도 콘텐츠가 똑같다"는 인식이 퍼지면 VIP 전환 유인이 약해짐.

### 3. 헬로우봇 — 월 9,900원 구독제
3강 중 유일한 구독 모델(챗봇 서비스 전체 접근권). "사주 콘텐츠는 고가 상품이 많다, 주 소비층은 27세 이상 직장인" vs "Z세대는 타로·심리테스트 무료 위주" — 연령/소비력에 따라 소비 패턴이 갈린다는 실증. 유료 타로 "그 사람의 속마음" 누적 조회수 100만+ — 개인화·감정적 콘텐츠는 유료라도 소비된다는 근거.

## 우리 앱(COSMOS_BEOMBOO)에 대한 시사점
현재 MVP는 상세 리포트(`report_screen.dart`)·심층 분석(`deep_dive`)까지 전부 무료 노출 중 — "지금 무료로 보여주는 것 중 어디까지를 유료 경계선으로 옮길지"가 핵심 결정 지점. 점신 사례는 "유료화하되 콘텐츠 차이가 없으면 불만만 쌓인다"는 반면교사, 헬로우봇 사례는 "개인화·감정적 공감 콘텐츠는 유료라도 소비된다"는 근거로 참고. **결제 로직 자체는 사람 결정 대기** — 이 문서는 결정 시점에 참고할 실측 자료.

## 정보구조 3가지 유형 (콘텐츠 심화 노출 방식)

| 유형 | 사례 | 구조 |
|---|---|---|
| **완전 분리형** | 포스텔러(메인 앱 vs "포스텔러 만세력" 별도 앱) | 캐주얼(밈·카드뉴스)은 메인, 십신·대운·세운·신살 등 전문 콘텐츠는 별도 앱. 별도 앱은 "7단계 활용법" 같은 자체 학습 가이드까지 갖춘 완결된 도구 |
| **단일 앱 점진 노출형** | 헬로우봇, 점신 | 앱을 안 나누고 대화형 UX/탭 추가로 심화 콘텐츠를 점차 노출. 헬로우봇은 챗봇 대화 흐름 안에서 건별 유료화, 점신은 기존 앱에 "만세력" 탭 추가 |
| **전문가 전용 단일 목적형** | 신운세(信운세) 만세력 | 캐주얼 레이어 없는 정통 명리 전문 도구. 자체 만세력 스펙 공개(자시 23:30~01:30 등) |

**3강 중 2곳(점신·헬로우봇)이 단일 앱형** — 우리 규모·구조(캐주얼 메인 앱 하나)에는 단일 앱 점진 노출형이 더 맞다. 이미 있는 "심층 분석(MBTI·관심사)" 확장 화면 패턴이 사실상 이 방식과 유사(메인 결과 화면은 가볍게 유지, 원하는 사람만 한 걸음 더 들어가는 별도 화면).
