# 결과 화면 구성 사례

## 결과 전달 방식 유형
1. **정적 리포트형**: 사주 명식(8글자) + 오행 그래프 + 텍스트 풀이 (전통 만세력 사이트 방식, 정확하지만 딱딱함)
2. **한 줄 요약 + 확장형**: "당신의 2026년 키워드는 OO" 헤드라인 → 탭하면 상세 풀이 펼쳐짐 (포스텔러류, SNS 공유 최적)
3. **대화형(챗봇)**: 캐릭터가 대화하듯 결과를 순차적으로 설명 (Dr.Saju 오픈소스 참고 — `chat` 라우트, 8종 캐릭터)
4. **카드/게임화형**: 스크래치나 카드 뒤집기로 결과를 하나씩 공개 (부적 카드 마케팅 사례 응용 가능)
5. **계산은 앱이, 해석은 외부 AI에 위임형 (2026-07-06 웹 검색 신규 발견)**: "만세력 천을귀인"(안드로이드 만세력 앱 1위, 누적 다운로드 40만)이 쓰는 방식 — 앱은 사주팔자·대운·세운·십신·신살 등 정밀 데이터만 계산해서 보여주고, "AI 사주풀이 프롬프트 생성" 기능으로 그 데이터를 담은 프롬프트를 자동으로 만들어준 뒤 사용자가 그걸 복사해 ChatGPT/Gemini 등 자신의 AI에 붙여넣어 해석받는 구조. 앱 자체는 AI API 연동·비용·응답 생성 책임을 전혀 안 지면서도 "AI로 깊이 있는 해석"이라는 가치는 제공하는 하이브리드 방식. 우리 앱의 "AI 해석 연동"이 아직 사람 결정 대기(API 키·비용·개인정보 전송 동의 이슈, PROJECT_ROUTER.md 참고)인 상황에서, 이 방식은 **우리가 API 비용을 전혀 들이지 않고도 심층 분석에 AI 해석을 붙일 수 있는 저비용 대안**으로 고려할 만함 — 다만 사용자가 직접 외부 AI 앱으로 복사해서 붙여넣어야 하는 번거로움(이탈 유발 가능)은 트레이드오프.

   **실제 프롬프트 템플릿 확인(2026-07-06, 네이버 블로그 실사용 후기)**: 이 방식을 실제로 쓰는 사용자가 올린 프롬프트 원문을 확인했다 — 놀랍도록 짧고 단순하다.
   > "저의 사주를 심층적으로 해석해 주세요. 태어난 날짜는 [년/월/일/오전 or 오후 00시 00분]이며, [경진년 정해월 경진일 갑오시]간지의 성별은 [여 or 남]입니다. 저의 성향, 강점, 약점, 타고난 기질, 성격적 특징을 상세히 분석해 주세요."

   즉 필요한 최소 정보는 ①생년월일시(양력) ②4기둥 60갑자 명식(한글, "년주 월주 일주 시주" 순으로 나열) ③성별, 딱 세 가지뿐이다. 작성자는 "챗지피티에게 명식을 캡쳐(이미지)해서 보내면 대운 방향도 못 읽고 간지 순서도 제대로 체크 못 한다"고 실제로 겪은 정확도 문제를 지적하며, **반드시 이미지가 아니라 이미 계산된 4기둥 텍스트를 그대로 프롬프트에 넣어야 한다**고 강조한다 — AI가 사주 계산 자체를 직접 하게 하면 안 되고, 계산은 앱(또는 우리 프로젝트의 `four_pillars.dart`)이 맡고 AI에는 결과값만 텍스트로 넘겨야 신뢰할 수 있다는 뜻. 이어서 "개인 성향 및 기질", "인간관계와 연애" 등 카테고리별 후속 질문 리스트도 함께 제공해, 첫 프롬프트 이후 사용자가 이어서 물어볼 질문까지 미리 안내하는 구조다.

   **우리 프로젝트에 대한 실질적 시사점**: 이 프롬프트에 필요한 데이터(4기둥 명식, 생년월일시, 성별)는 이미 `share_text.dart`의 `buildShareText`가 거의 동일한 형태로 조합하고 있다 — 즉 이 패턴을 실제로 도입한다면 완전히 새로 만들 필요 없이, 기존 공유 텍스트 조립 로직을 살짝 변형해 "AI에게 물어보기" 버튼(예: 프롬프트 텍스트를 클립보드에 복사) 하나만 추가하는 정도로 저비용 프로토타입이 가능해 보인다. 다만 이는 구현 여부를 결정하는 게 아니라(AI 해석 연동은 여전히 사람 결정 대기), 결정 시점에 바로 참고할 구체적 설계 힌트로 남겨두는 것이다.

## 공통적으로 포함되는 결과 섹션
- 사주 명식 요약 (년/월/일/시주, 오행 분포 시각화 — 도넛차트/막대그래프 형태가 흔함)
- 성격/기질 분석
- 애정운 / 재물운 / 건강운 / 직업운 등 카테고리별 운세
- 오늘·이번달·올해 흐름 (시계열 운세)
- 궁합 보기로 자연스럽게 연결되는 CTA

## SNS 공유용 결과 카드 디자인 포인트
- 9:16 세로 비율(인스타 스토리 규격)로 별도 공유 이미지 생성
- 텍스트는 한 줄 임팩트 카피 중심, 부가 설명은 최소화
- 앱 로고/워터마크를 결과 카드 하단에 작게 배치 (바이럴 시 유입 경로 확보)

## 무료/유료 경계 설계 (전환 구조)
- 무료: 기본 사주 명식 + 짧은 요약 풀이
- 유료: 200쪽 이상 상세 리포트, PDF 다운로드, 전문가 상담 연결 (포스텔러 사례)
- 결과 화면 안에서 "더 알아보기" 형태로 유료 리포트 미리보기 노출 → 전환 유도

## 무료/유료 경계 실측 비교 — 3강 앱은 서로 다른 모델을 쓴다 (2026-07-06 웹 검색 보강)

경쟁 앱 3사(포스텔러·점신·헬로우봇)의 실제 후기·리뷰·인터뷰를 조사해보니, "무료는 기본, 유료는 심화"라는 한 줄 요약과 달리 **수익화 철학 자체가 셋이 서로 다르다.** 우리 앱의 결제 로직은 아직 사람 결정 대기 중(PROJECT_ROUTER.md 참고)이지만, 어떤 모델을 고르든 참고할 수 있도록 세 모델을 구체적으로 정리해둔다.

### 1. 포스텔러 — 캐시(포스) 기반 콘텐츠 단위 결제
- 실제 결제 단가: "500포스(약 5,000원)" 충전, 첫 결제 시 1+1 이벤트로 사실상 반값 유도. 궁합 프리미엄 리포트는 22,000원 사례도 확인됨(네이버 블로그 실사용 후기, 2025-01-22).
- 무료/유료 콘텐츠 차이가 분명함(무료: 짧은 요약, 유료: 개인화된 심층 풀이) — 결제 자체에 대한 저항감이 있고, 실사용자 후기에서 "운세 콘텐츠 단가가 좀 있는 편(커피 한 잔 값 순삭)"이라는 표현이 반복 발견됨.
- 다만 콘텐츠 볼륨/퀄리티에 대한 기대와 실제 산출물 사이 괴리도 보고됨: AI 상담 캐릭터 상담이 "생각보다 내용이 길진 않았다"는 후기(하이퍼골드 블로그, 2026-02-10) — 유료 결제 후 기대 대비 짧다고 느끼면 재구매 의사가 낮아질 수 있음을 시사.
- 출처: [포스텔러 AI 상담소 솔직 후기](https://hyper-info.co.kr/60)(2026-02-10), [네이버 블로그 실사용 후기](https://m.blog.naver.com/cars_a/223730022863)(2025-01-22)

### 2. 점신 — 광고 기반 완전 무료 + VIP는 "광고 제거"가 사실상 전부
- 점신 개발자가 직접 리뷰에 답변한 내용(Google Play, 2025 기준): **"무료와 유료버전은 광고의 노출 유무 외에는 현재 동일한 기능을 갖고 있습니다"** — 즉 콘텐츠 자체는 무료/유료가 거의 동일하고, 결제의 실질 가치는 광고 제거뿐.
- 대신 광고 강도에 대한 불만이 매우 많음: "30초짜리 광고 하나 뜨고 연달아 15초짜리 광고", "X표시가 늦게 나오거나 누르는 순간 광고 페이지로 순간이동" 등 광고 UX 자체가 주요 불만 소재(App Store 리뷰, 4.7점 평점에도 불구하고 반복 지적).
- 시사점: 광고 기반 모델은 무료 이용자 규모(1,800만+)를 극대화하기 좋지만, "결제해도 콘텐츠가 똑같다"는 인식이 퍼지면 VIP 전환 유인이 약해짐 — 유료 전환을 원한다면 광고 제거 이상의 실질적 콘텐츠 차별화가 필요하다는 반례로 참고할 만함.
- 출처: Google Play 앱 리뷰 및 개발자 답변, App Store 리뷰(2026 점신)

### 3. 헬로우봇 — 월 9,900원 구독제(챗봇 콘텐츠)
- 유일하게 구독(월정액) 모델을 쓰는 3강 앱. 개별 콘텐츠 결제가 아니라 챗봇 서비스 전체 접근권을 파는 방식.
- 띵스플로우 콘텐츠 리드 인터뷰(미디어오늘, 2021-06-12)에 따르면: **"사주 콘텐츠는 전문성을 바탕으로 한 고가 상품이 많다. 주 소비자층은 27세 이상 직장인"** — 반면 **"Z세대는 사주보다는 가볍게 볼 수 있는 타로나 심리테스트... 무료 사주 콘텐츠 이용률이 높다"**. 즉 연령/소비력에 따라 사주(고가 지불 의향 있음) vs 타로·심리테스트(무료 소비 위주)로 소비 패턴이 갈린다는 실증적 근거.
- 유료 콘텐츠도 실제로 잘 팔린 사례 있음: "그 사람의 속마음"(유료 타로)이 누적 조회수 100만 이상 — 개인화되고 감정적으로 와닿는 콘텐츠라면 유료라도 소비된다는 근거.
- 출처: [미디어오늘 인터뷰 "왜 타로·사주 챗봇을 월 9900원에 구독할까"](https://www.mediatoday.co.kr/news/articleView.html?idxno=213775)(2021-06-12)

### 우리 앱(COSMOS_BEOMBOO)에 대한 시사점 (참고용, 최종 결정은 보류 상태)
- 현재 우리 MVP는 상세 리포트(`report_screen.dart`)와 심층 분석(`deep_dive` — MBTI·관심사)까지 전부 무료로 노출 중 — 위 3개 모델 중 어느 쪽으로 가든, "지금 무료로 보여주는 것 중 어디까지를 유료 경계선으로 옮길지"가 핵심 결정 지점이 될 것.
- 점신의 사례는 "유료화하되 콘텐츠 차이가 없으면 불만만 쌓인다"는 반면교사로, 헬로우봇의 사례는 "개인화·감정적 공감이 강한 콘텐츠는 유료라도 소비된다"는 근거로 함께 참고할 만함.
- 결제 로직 자체는 여전히 사람 결정 대기 항목(PROJECT_ROUTER.md의 "심층 분석" 행 참고) — 이 절은 그 결정을 대신하지 않고, 결정 시점에 참고할 실측 자료를 미리 쌓아두는 목적.
