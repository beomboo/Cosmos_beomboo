# UI/UX 트렌드 — 젊은 감성의 사주 앱

## 핵심 방향: "무거운 점술" → "가벼운 자기이해 콘텐츠"
전통 사주 상담소의 무거운 분위기를 벗고, 모바일에 최적화된 직관적이고 재미있는 경험으로 재해석하는 것이 MZ세대 공략의 핵심.

## 1. 캐릭터화 (Characterization)
- MZ세대는 무겁고 진지한 점술보다 **"캐릭터화된 운세"**를 선호
- MBTI처럼 "나는 어떤 동물/기운 타입일까?" 식의 **라벨링 게임 형식**이 효과적
- 예: 최고심 브랜드의 스크래치 복권형 부적 카드 — "돈벼락", "합격" 같은 직관적 문구 + 귀여운 일러스트
- 카카오·토스 등 대형 플랫폼도 자체 캐릭터로 디지털 부적 제작해 신년 마케팅 활용

## 2. 컬러 팔레트
- 밝은 원색보다 **소프트 톤(파스텔) 선호도가 높음** — 사용자 후기에서도 "부드러운 색조가 더 낫다" 언급
- Z세대 팔레트 조합 예시:
  - 피부/베이스: 피치, 소프트 핑크
  - 포인트(헤어/그래픽): 라벤더, 민트 그린
  - 서브: 세이지 그린, 페리윙클
  - 강조(CTA/뱃지): 코랄, 터쿼이즈

## 3. 언어/카피 톤
- 명리학 전문 용어(십신, 편관 등) 그대로 노출 금지 — 일상어로 재해석해서 전달
- "내 2026년 키워드는 OO" 처럼 **한 줄 요약형 카피**가 SNS 공유에 효과적

## 4. SNS 바이럴 설계
- 결과 화면을 **인스타그램 스토리 규격(9:16)에 최적화**해서 캡처/공유하기 좋게 디자인
- 공유 버튼을 결과 화면 상단/CTA 위치에 배치 (하단에 묻히지 않게)

## 5. 인터랙션 요소
- 로딩(계산 중) 구간에 캐릭터 애니메이션 배치 — 대기 시간을 콘텐츠화
- 대화형 인터페이스(AI 챗봇 형태)로 결과를 전달하는 방식도 트렌드 (예: Dr.Saju의 8종 캐릭터 챗)
- 스크래치/카드 뒤집기 같은 게임화 인터랙션으로 결과 공개 연출

## 6. 온보딩 vs 스토어 스크린샷 구분
- 스토어 스크린샷은 "설치 유도"가 목적, 온보딩 화면은 "서비스 이해·기능 활용"이 목적 — 서로 다른 메시지로 설계해야 함

## 참고 경쟁 앱 UI 포인트
- **포스텔러**: 캐릭터 기반 인터페이스, 무료 콘텐츠 진입장벽 최소화. 사용자 리뷰 "캐릭터들도 진짜 귀엽고 앱이 전체적으로 사랑스럽다"
- **점신**: '운세보고서' 형식으로 오늘의 운세·총운·시간대별 운세·바이오리듬·추천소식을 한 화면 리포트로 종합. 상담사 랭킹으로 신뢰 부여
- **헬로우봇**: iOS 11 이후 디자인 철학을 충실히 따름 — **흰 배경 + 굵은 글씨 + 둥근 카드**. 캐릭터 '라마마' 등이 탄탄한 세계관을 가지고 챗봇 형태로 대화하듯 결과 전달(판빙빙 등 유명인 캐릭터 콜라보 사례도 있음)
- **Dr.Saju(오픈소스)**: 8가지 개성 캐릭터와의 AI 챗 형태로 결과 전달 (`chat/[characterId]` 라우트) — 결과를 텍스트 나열이 아닌 대화로 푸는 방식 참고할 만함

## 디자인 방향성 두 갈래 — 실제 소스 코드로 확인된 대비
경쟁작/레퍼런스를 직접 뜯어본 결과, 시장에는 뚜렷이 다른 두 스타일이 공존합니다. 우리 앱의 톤을 정할 때 이 축을 기준으로 결정하는 것을 권장합니다.

| | **파스텔 큐트형** (포스텔러·헬로우봇) | **다크 프리미엄 미스틱형** (Dr.Saju 실제 코드 기준) |
|---|---|---|
| 배경 | 흰색/밝은 배경 | `#0a0a0f` 거의 검정 |
| 포인트 컬러 | 파스텔(피치·라벤더·민트) | 보라(violet) `#8b5cf6` + 골드 `#C5A44E` |
| 톤 | 귀엽고 가벼움, 밈/게임화 | 신비롭고 고급스러움, "운명을 들여다보는" 무게감 |
| 캐릭터 | 귀여운 마스코트 중심 | 캐릭터별 챗 라우트는 있지만 비주얼은 절제된 다크 UI |
| 타겟 인상 | 10대~20대 초반, MZ 밈문화 친화 | 20대 후반~30대, "믿을 만한 프리미엄 서비스" 인상 |
| 참고 근거 | 사용자 리뷰, 캐릭터 마케팅 사례 | `globals.css` 실측 (07_오픈소스_계산로직 참고) |

→ "이해하기 쉽고 젊은 느낌"이라는 요청 사항은 두 방향 모두로 해석 가능하므로, 무드보드 단계에서 실제 컬러 조합 시안 2종(파스텔/다크)을 비교해보는 것을 제안합니다.
