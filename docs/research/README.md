## 핵심 요약

**시장**: 한국 운세 시장 약 1.4조 원, 연 25% 성장. 포스텔러가 900만 가입자로 압도적 1위이며 이용자 80% 이상이 2030세대. 점신(1,500만 다운로드), 헬로우봇(다운로드 1위, 평점 4.7)이 뒤를 잇는 3강 구도.

**성공 공식**: 어려운 명리학 용어를 일상어/밈으로 재해석 + 캐릭터 기반 친근한 UI + 무료로 낮춘 진입장벽 + 유료 심화 리포트 전환 + SNS 공유 최적화.

**콘텐츠 톤 (실측)**: 포스텔러 인스타 최근 게시물 분석 결과, 명리학 개념을 "무자일주가 단톡에서 읽씹하는 이유"처럼 완전히 밈화해서 소비시키는 전략이 반응이 가장 좋음(좋아요 700~1,000+). 결과 화면 카피도 이 정도 캐주얼함을 목표로 삼을 만함.

**입력 설계**: 생년월일시·양력음력·성별이 필수, 출생지는 선택. "태어난 시간 모름" 케이스를 우아하게 처리하는 것이 이탈률에 직결.

**무료/유료 경계 (2026-07-06 웹 검색 보강)**: 3강 앱이 서로 다른 수익화 철학을 씀 — 포스텔러는 캐시(포스) 기반 콘텐츠 단위 결제(500포스≈5,000원, 궁합 프리미엄 22,000원 사례), 점신은 광고 기반 완전 무료로 VIP도 사실상 "광고 제거"만 다름(개발자가 직접 확인), 헬로우봇은 월 9,900원 구독제. 헬로우봇 인터뷰에 따르면 사주는 27세 이상 직장인이 주 소비층(고가 지불 의향 있음)이고 Z세대는 무료 콘텐츠·타로/심리테스트 위주로 소비 — **갤럽 설문조사로 교차 검증 완료**: 유료 경험자 여성 58% vs 남성 22%(20대 남성은 8%뿐), 16~24세 여성 유료 비중은 2019~2022년 계속 하락 추세(SBS 프리미엄). 자세한 내용은 [05_결과화면_사례](05_결과화면_사례/결과화면_사례.md)의 "무료/유료 경계 실측 비교" 절과 [02_경쟁앱_분석](02_경쟁앱_분석/경쟁앱_분석.md)의 "유료 소비 성별·연령 실측 데이터" 절 참고.

**2026년 최신 시장 지표 (2026-07-06 보강)**: 점신 누적 다운로드 1,700만·2025-12 MAU 79만(2030 비중 65.6%), 포스텔러 가입자 750만·MAU 63만(2030 비중 57%). 유료 가격대는 990원 소액~10만원대 종합 패키지까지 폭넓음. 비대면 전문가 상담(천명 등)도 신규 트렌드로 주류화(신규 상담 절반 이상 비대면). 자세한 내용은 [02_경쟁앱_분석](02_경쟁앱_분석/경쟁앱_분석.md)의 "2026년 최신 시장 지표" 절 참고.

**신규 결과 전달 방식 발견(2026-07-06) — "계산은 앱, 해석은 외부 AI"**: 만세력 1위 앱 "만세력 천을귀인"(다운로드 40만)이 사주 데이터를 정밀 계산해주고 "AI 프롬프트 생성" 기능으로 ChatGPT/Gemini에 붙여넣을 프롬프트를 자동 작성해주는 방식을 씀 — 앱이 AI API 비용·연동을 전혀 안 지면서도 AI 해석 경험을 제공하는 하이브리드 모델. **실제 프롬프트 원문까지 확인**: 필요한 최소 정보는 생년월일시+4기둥 명식(텍스트, 이미지 아님)+성별뿐인 매우 짧은 템플릿 — 우리 `share_text.dart`의 `buildShareText`가 이미 거의 같은 데이터를 조합하고 있어 이 패턴을 도입한다면 저비용으로 프로토타입 가능해 보임. 우리 앱의 "AI 해석 연동" 사람 결정 대기 항목에 저비용 대안으로 고려할 만함(트레이드오프: 사용자가 직접 외부 앱에 복사해야 하는 번거로움). 자세한 내용은 [05_결과화면_사례](05_결과화면_사례/결과화면_사례.md)의 "결과 전달 방식 유형" 5번 항목 참고.

**해석 단계의 학파 편차 — 지장간·용신 (2026-07-06)**: 지금까지 확인한 절기·자시 편차는 "계산 방식"의 차이였는데, 지장간(支藏干)·용신(用神)은 **해석 단계에서 갈리는 별개의 편차**임을 확인 — 지장간 기준 자체가 문헌마다 다르고(연해자평이 대표적 기준), 오행 개수 집계도 지장간 포함 여부로 갈리며, 용신 판단법도 억부법/조후법 최소 두 갈래가 서로 다른 답을 냄. 우리 `ohaengCount`는 지장간 미포함(표면 8글자만)이라 "초보자 추천" 방식과 이미 일치함을 확인. 용신 개념 자체는 아직 우리 앱에 없음(현재는 "우세 오행"만 다룸 — 서로 다른 개념이니 향후 확장 시 혼동 주의). 자세한 내용은 [01_사주팔자_기초개념](01_사주팔자_기초개념/기초개념_정리.md)의 "지장간·용신 판단법" 절 참고.

**기술 스택 제안**: 계산 엔진은 `manseryeok`(TypeScript) 채택, 해석은 AI가 담당하는 "계산-해석 분리" 구조. Dr.Saju의 실제 플로우(`landing→input→preview→result→report`, `share/referral/upsell→payment`)가 온보딩부터 과금까지 이어지는 전체 사용자 여정 설계에 그대로 참고 가능.

**콘텐츠 갭 발견(2026-07-06) — 십신(十神)이 우리 앱에 없음**: `project/lib/` 전체 검색 결과 십신 관련 로직이 전혀 구현돼 있지 않음을 확인. 오행 분포와 함께 명리학의 양대 해석 축인데 우리는 오행만 다루고 있음. 웹 검색으로 "10개 그대로 노출 대신 5그룹(비겁/식상/재성/관성/인성)으로 단순화 + 실생활 문장" 실제 사례를 찾음 — 일간 기준 기계적 계산이라 새 라이브러리/AI 연동 없이도 1단계(규칙 기반)로 구현 가능해 보임(심층 분석 MBTI 때와 비슷한 결). **추가 확인**: 포스텔러도 캐주얼 메인 앱(밈스러운 SNS 콘텐츠)과 별도로 "포스텔러 만세력"이라는 전문가용 서브 앱을 따로 두고, 거기서만 십신을 "열 개의 별 차트"+용어 해설 형태로 노출함 — 우리가 이미 제안한 "심층 분석처럼 별도 확장 화면에 배치"가 시장 1위 사업자의 실제 정보구조 전략과 일치함을 재확인. **정보구조 3유형 비교 완료**: 포스텔러(완전 분리형 두 앱) vs 점신·헬로우봇(둘 다 단일 앱 안에서 점진적으로 깊이 들어가는 방식 — 점신은 앱 업데이트로 "만세력" 탭을 기존 앱에 추가, 헬로우봇은 챗봇 대화로 깊이 들어가며 건별 유료화) vs 신운세(캐주얼 레이어 없는 전문가 전용 단일 목적 도구) — **3강 중 2곳이 단일 앱형이라 이 방향이 시장 다수의 선택**임을 확인, 우리 규모에도 단일 앱형(이미 있는 심층 분석 확장 화면 패턴과 유사)이 더 맞아 보임. 신운세 조사 중 자시 시작 시각조차 23:00/23:30 두 값이 실제로 쓰인다는 것도 추가로 확인(영향 범위 30분 이내로 좁아 기존 세 이슈보다 우선순위 낮음). 자세한 내용은 [01_사주팔자_기초개념](01_사주팔자_기초개념/기초개념_정리.md)의 "콘텐츠 갭 발견"·"심층 콘텐츠 정보구조 3가지 유형 비교" 절 참고. (구현 여부·우선순위는 결정 대기 — 단순 콘텐츠 아이디어 기록)

**용신(用神) 콘텐츠 갭 — 노출 방식 두 진영 확인(2026-07-07)**: 십신과 마찬가지로 우리 앱에 없는 용신도, 실제 경쟁 앱 검색 결과 뚜렷하게 두 진영으로 갈림을 확인 — "사주명"·"개운사주-격국용신분석"·"사주팔자(Sia Makerlab)"는 "용신" 용어를 그대로 노출(개운사주는 억부론/조후론/격국론 세 갈래를 각각 별도 기능으로 나란히 제공해 학파 편차에 정면 대응)하는 반면, "사주한눈에"·"데이사주"는 "용신"이란 단어 자체를 아예 안 쓰고 "오행 라이프 코칭"/"개운법"(부족한 오행에 맞는 색상·음식·활동 추천)으로 완전히 순화해서 보여줌. 우리 앱은 캐주얼 톤이라 후자 방향이 더 맞고, 이미 있는 오행 밸런스 기능에 자연스럽게 이어붙일 수 있어 십신보다 구현 문턱이 낮아 보임. 자세한 내용은 [01_사주팔자_기초개념](01_사주팔자_기초개념/기초개념_정리.md)의 "용신(用神) 콘텐츠 갭 — 경쟁 앱의 노출 방식 실측" 절 참고. (구현 여부는 여전히 사람 결정 대기 — 단순 콘텐츠 아이디어 기록)

## Agent-Reach 채널 현황 (2026-07-04 기준, `agent-reach doctor --json`; 2026-07-06에 실제 호출로 재검증)

이번 리서치에서 실제로 쓴 채널은 위 "주요 수집 채널" 열(웹 검색·GitHub·Instagram)이 전부지만, Agent-Reach 자체는 총 15개 채널을 지원한다. 아래는 현재 이 환경에서 각 채널이 실제로 작동하는지 점검한 결과이며, 향후 추가 리서치가 필요할 때 어떤 채널을 바로 쓸 수 있는지 참고용으로 남겨둔다.

**⚠️ 2026-07-06 실측 정정**: `agent-reach doctor`가 🟢로 보고하는 것은 "설정/바이너리가 설치돼 있다"는 뜻일 뿐, 실제 호출 성공을 보장하지 않는다. `opencli twitter search`, `opencli reddit search`를 이 자동화 세션에서 직접 실행해보니 **둘 다 `BROWSER_CONNECT`(Browser Bridge 확장 프로그램 미연결) 오류로 실패**했다 — OpenCLI 백엔드를 쓰는 채널(Twitter/X·Reddit·Facebook·Instagram·B站·샤오홍슈 전부)은 사람이 상시 켜둔 Chrome + OpenCLI 확장 프로그램이 필요한데, 이 무인 자동화 세션에는 그런 브라우저가 없다. 아래 표의 🟢는 "설정상 가능"일 뿐 "이 세션에서 바로 호출 가능"이 아니므로, 실제로 매번 쓸 수 있다고 확인된 채널은 **GitHub(`gh`)·전역 시맨틱 검색(Exa)·임의 웹페이지(Jina Reader)·YouTube(yt-dlp)·RSS 뿐**이다.

| 채널 | doctor 보고 상태 | 2026-07-06 실측 | 비고 |
|---|---|---|---|
| GitHub 저장소/코드 | 🟢 사용 가능 | ✅ 실제 작동 확인(이번 세션에서 다수 사용) | gh CLI, 읽기·검색·API 완전 지원 |
| 전역 시맨틱 검색 | 🟢 사용 가능 | ✅ 실제 작동 확인(이번 세션에서 다수 사용) | Exa(mcporter 경유), API 키 불필요 |

| Reddit | 🟢 사용 가능(doctor 기준) | ❌ `BROWSER_CONNECT` 오류로 실패 | OpenCLI — 위와 동일 |
