# 사주팔자 앱 리서치 자료

**이 디렉터리는 [Agent-Reach](https://github.com/Panniantong/Agent-Reach)로 수집한 원자료(raw material) 전용입니다.** 웹 시맨틱 검색, GitHub 코드/저장소 검색, Instagram 게시물 조회 등 여러 채널에서 모은 내용을 주제별로 정리해뒀습니다. 이 자료를 바탕으로 실제로 만든 UI 목업은 이 디렉터리가 아니라 루트의 [`/mockups`](../mockups/README.md)에 별도로 있습니다 — **`research/`는 자료 수집, `mockups/`는 그 자료를 소화해 만든 산출물**이라는 역할 구분입니다.

## 왜 나뉘어 있는가
- `research/*.md` — Agent-Reach가 가져온 정보를 사람이 읽기 좋게 정리한 노트. 언제든 다시 조사해서 갱신될 수 있는 "재료"
- `../mockups/*.html` — 그 재료를 바탕으로 판단·선택이 들어간 "완성물". 개발 시 UI/UX 기준으로 그대로 쓰임

## 핵심 요약

**시장**: 한국 운세 시장 약 1.4조 원, 연 25% 성장. 포스텔러가 900만 가입자로 압도적 1위이며 이용자 80% 이상이 2030세대. 점신(1,500만 다운로드), 헬로우봇(다운로드 1위, 평점 4.7)이 뒤를 잇는 3강 구도.

**성공 공식**: 어려운 명리학 용어를 일상어/밈으로 재해석 + 캐릭터 기반 친근한 UI + 무료로 낮춘 진입장벽 + 유료 심화 리포트 전환 + SNS 공유 최적화.

**콘텐츠 톤 (실측)**: 포스텔러 인스타 최근 게시물 분석 결과, 명리학 개념을 "무자일주가 단톡에서 읽씹하는 이유"처럼 완전히 밈화해서 소비시키는 전략이 반응이 가장 좋음(좋아요 700~1,000+). 결과 화면 카피도 이 정도 캐주얼함을 목표로 삼을 만함.

**입력 설계**: 생년월일시·양력음력·성별이 필수, 출생지는 선택. "태어난 시간 모름" 케이스를 우아하게 처리하는 것이 이탈률에 직결.

**기술 스택 제안**: 계산 엔진은 `manseryeok`(TypeScript) 채택, 해석은 AI가 담당하는 "계산-해석 분리" 구조. Dr.Saju의 실제 플로우(`landing→input→preview→result→report`, `share/referral/upsell→payment`)가 온보딩부터 과금까지 이어지는 전체 사용자 여정 설계에 그대로 참고 가능.
