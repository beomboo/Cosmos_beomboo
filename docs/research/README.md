## 핵심 요약

**시장**: 한국 운세 시장 약 1.4조 원, 연 25% 성장. 포스텔러가 900만 가입자로 압도적 1위이며 이용자 80% 이상이 2030세대. 점신(1,500만 다운로드), 헬로우봇(다운로드 1위, 평점 4.7)이 뒤를 잇는 3강 구도.

**성공 공식**: 어려운 명리학 용어를 일상어/밈으로 재해석 + 캐릭터 기반 친근한 UI + 무료로 낮춘 진입장벽 + 유료 심화 리포트 전환 + SNS 공유 최적화.

**콘텐츠 톤 (실측)**: 포스텔러 인스타 최근 게시물 분석 결과, 명리학 개념을 "무자일주가 단톡에서 읽씹하는 이유"처럼 완전히 밈화해서 소비시키는 전략이 반응이 가장 좋음(좋아요 700~1,000+). 결과 화면 카피도 이 정도 캐주얼함을 목표로 삼을 만함.

**입력 설계**: 생년월일시·양력음력·성별이 필수, 출생지는 선택. "태어난 시간 모름" 케이스를 우아하게 처리하는 것이 이탈률에 직결.

**무료/유료 경계 (2026-07-06 웹 검색 보강)**: 3강 앱이 서로 다른 수익화 철학을 씀 — 포스텔러는 캐시(포스) 기반 콘텐츠 단위 결제(500포스≈5,000원, 궁합 프리미엄 22,000원 사례), 점신은 광고 기반 완전 무료로 VIP도 사실상 "광고 제거"만 다름(개발자가 직접 확인), 헬로우봇은 월 9,900원 구독제. 헬로우봇 인터뷰에 따르면 사주는 27세 이상 직장인이 주 소비층(고가 지불 의향 있음)이고 Z세대는 무료 콘텐츠·타로/심리테스트 위주로 소비 — **갤럽 설문조사로 교차 검증 완료**: 유료 경험자 여성 58% vs 남성 22%(20대 남성은 8%뿐), 16~24세 여성 유료 비중은 2019~2022년 계속 하락 추세(SBS 프리미엄). 자세한 내용은 [05_결과화면_사례](05_결과화면_사례/결과화면_사례.md)의 "무료/유료 경계 실측 비교" 절과 [02_경쟁앱_분석](02_경쟁앱_분석/경쟁앱_분석.md)의 "유료 소비 성별·연령 실측 데이터" 절 참고.

**2026년 최신 시장 지표 (2026-07-06 보강)**: 점신 누적 다운로드 1,700만·2025-12 MAU 79만(2030 비중 65.6%), 포스텔러 가입자 750만·MAU 63만(2030 비중 57%). 유료 가격대는 990원 소액~10만원대 종합 패키지까지 폭넓음. 비대면 전문가 상담(천명 등)도 신규 트렌드로 주류화(신규 상담 절반 이상 비대면). 자세한 내용은 [02_경쟁앱_분석](02_경쟁앱_분석/경쟁앱_분석.md)의 "2026년 최신 시장 지표" 절 참고.

**신규 결과 전달 방식 발견(2026-07-06) — "계산은 앱, 해석은 외부 AI"**: 만세력 1위 앱 "만세력 천을귀인"(다운로드 40만)이 사주 데이터를 정밀 계산해주고 "AI 프롬프트 생성" 기능으로 ChatGPT/Gemini에 붙여넣을 프롬프트를 자동 작성해주는 방식을 씀 — 앱이 AI API 비용·연동을 전혀 안 지면서도 AI 해석 경험을 제공하는 하이브리드 모델. **실제 프롬프트 원문까지 확인**: 필요한 최소 정보는 생년월일시+4기둥 명식(텍스트, 이미지 아님)+성별뿐인 매우 짧은 템플릿 — 우리 `share_text.dart`의 `buildShareText`가 이미 거의 같은 데이터를 조합하고 있어 이 패턴을 도입한다면 저비용으로 프로토타입 가능해 보임. 우리 앱의 "AI 해석 연동" 사람 결정 대기 항목에 저비용 대안으로 고려할 만함(트레이드오프: 사용자가 직접 외부 앱에 복사해야 하는 번거로움). 자세한 내용은 [05_결과화면_사례](05_결과화면_사례/결과화면_사례.md)의 "결과 전달 방식 유형" 5번 항목 참고.

**기술 스택 제안**: 계산 엔진은 `manseryeok`(TypeScript) 채택, 해석은 AI가 담당하는 "계산-해석 분리" 구조. Dr.Saju의 실제 플로우(`landing→input→preview→result→report`, `share/referral/upsell→payment`)가 온보딩부터 과금까지 이어지는 전체 사용자 여정 설계에 그대로 참고 가능.

**콘텐츠 갭 발견(2026-07-06) — 십신(十神)이 우리 앱에 없음**: `project/lib/` 전체 검색 결과 십신 관련 로직이 전혀 구현돼 있지 않음을 확인. 오행 분포와 함께 명리학의 양대 해석 축인데 우리는 오행만 다루고 있음. 웹 검색으로 "10개 그대로 노출 대신 5그룹(비겁/식상/재성/관성/인성)으로 단순화 + 실생활 문장" 실제 사례를 찾음 — 일간 기준 기계적 계산이라 새 라이브러리/AI 연동 없이도 1단계(규칙 기반)로 구현 가능해 보임(심층 분석 MBTI 때와 비슷한 결). **추가 확인**: 포스텔러도 캐주얼 메인 앱(밈스러운 SNS 콘텐츠)과 별도로 "포스텔러 만세력"이라는 전문가용 서브 앱을 따로 두고, 거기서만 십신을 "열 개의 별 차트"+용어 해설 형태로 노출함 — 우리가 이미 제안한 "심층 분석처럼 별도 확장 화면에 배치"가 시장 1위 사업자의 실제 정보구조 전략과 일치함을 재확인. **정보구조 3유형 비교 완료**: 포스텔러(완전 분리형 두 앱) vs 헬로우봇(단일 앱 안에서 챗봇 대화로 점진적으로 깊이 들어가며 건별 유료화, 만세력도 "초보자 친화적"으로 순화해 통합) vs 신운세(캐주얼 레이어 없는 전문가 전용 단일 목적 도구) — 우리 규모엔 헬로우봇식(이미 있는 심층 분석 확장 화면 패턴과 유사)이 더 맞아 보임. 신운세 조사 중 자시 시작 시각조차 23:00/23:30 두 값이 실제로 쓰인다는 것도 추가로 확인(영향 범위 30분 이내로 좁아 기존 세 이슈보다 우선순위 낮음). 자세한 내용은 [01_사주팔자_기초개념](01_사주팔자_기초개념/기초개념_정리.md)의 "콘텐츠 갭 발견"·"심층 콘텐츠 정보구조 3가지 유형 비교" 절 참고. (구현 여부·우선순위는 결정 대기 — 단순 콘텐츠 아이디어 기록)

**절기(節氣) 정밀 계산 (2026-07-06 GitHub 소스코드 분석)**: 우리 앱의 현재 근사(입춘=2/4 고정, 월 경계=그레고리력 월)를 개선할 실제 경로를 찾음 — manseryeok는 Jean Meeus의 저정밀 태양 위치 공식(순수 삼각함수, 외부 의존성 없음, 정확도 약 0.01°/분 단위)으로 절기를 계산하고, KASI 기준 보정표(1800~2300년)로 추가 보정함. **Meeus 공식만 Dart로 포팅해도** 지금의 "며칠" 오차를 "몇 분" 수준으로 줄일 수 있어, 실제 정밀도 개선 시 가장 먼저 시도할 만한 작은 단위 작업. 자세한 내용은 [07_오픈소스_계산로직](07_오픈소스_계산로직/오픈소스_라이브러리.md)의 "manseryeok의 절기 계산 알고리즘 상세 분석" 절 참고.

**자시(子時) 관법 — 절기와 별개인 두 번째 정확도 이슈 (2026-07-06)**: manseryeok 소스코드 확인 중 자시(23:00~23:59) 출생을 처리하는 3가지 관법(`midnight`/`splitJasi`/`jasi`)이 실제로 존재함을 확인. 우리 앱은 그중 가장 단순한 `midnight`(자시 롤오버 없음)에 해당하는데, 이는 그동안 코드 어디에도 명시적으로 문서화돼 있지 않았던 선택이었음(절기 근사 한계만 문서화돼 있었음). 전통 명리학에서 더 흔한 `splitJasi`(조자시/야자시 분리)와는 23시대 출생자의 시주가 달라질 수 있음. 자세한 내용은 [04_입력정보_요구사항](04_입력정보_요구사항/입력정보_요구사항.md)의 "자시 처리 3가지 관법 실측" 절 참고.

**세 번째 정확도 이슈(2026-07-06) — 진태양시(AST) 미보정**: 포스텔러 만세력이 "야자시/조자시"를 사용자 선택 옵션으로 제공한다는 걸 확인해 이 이슈가 시장 표준 앱조차 강제하지 않는 실제 유파 차이임을 재확인. 더불어 `korean-saju`(PyPI) 패키지 문서에서 "일주·시주는 KST가 아니라 진태양시(AST, 경도 보정) 캘린더 날짜를 기준으로 해야 한다"는 경고를 확인했는데, 우리 코드는 `BirthInfo.birthPlace`(출생지)를 수집하고도 실제 계산(`four_pillars.dart`)에는 전혀 쓰지 않아 AST 보정이 아예 없음을 확인 — 절기 근사·자시 관법에 이은 세 번째 정확도 항목. **구체적 크기까지 확인**: 서울 기준 경도 보정 약 30~32분(고정) + 균시차 계절 변동 최대 ±16분 = 총 -14분~-48분 범위 — 시진 경계(2시간 단위) 부근 출생자는 이 보정만으로 실제 시주 자체가 바뀔 수 있는 실질적 문제. 자세한 내용은 [07_오픈소스_계산로직](07_오픈소스_계산로직/오픈소스_라이브러리.md)의 "자시 관법·진태양시 보정"·"진태양시 미보정의 실제 크기" 절 참고.
