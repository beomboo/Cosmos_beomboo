# 공망(空亡) — 콘텐츠 갭

_최종 확인: 2026-07-18_

**현황**: 미구현. 계산 규칙 자체는 아래처럼 공식·대응표로 완전히 특정됐으나(2026-07-18 보강), 공망은 신살과 마찬가지로 게이트 항목(십신·용신·대운/세운·십이운성·신살·공망 등 6종 콘텐츠 갭)이라 실제 구현 여부는 여전히 사람 결정 대기.

## 개념 — 순중공망(旬中空亡)의 원리
천간은 10개, 지지는 12개라서 갑(甲)부터 순서대로 짝지어 나가면 10개 천간이 다 소진된 시점에 지지는 아직 2개가 남는다. 이렇게 60갑자를 10개씩 묶은 6개의 "순(旬)"(갑자순·갑술순·갑신순·갑오순·갑진순·갑인순) 각각에서, 그 순에 속한 10개 간지가 다 쓰이고도 짝을 이루지 못해 "비어버린(空)" 지지 2개가 바로 그 순 전체의 **공망**이다. 즉 공망은 절기·명식 개인차와 무관하게 **일주(또는 다른 기둥)가 어느 순에 속하는지만으로 정해지는 고정 배치** — 십이운성·12신살처럼 "기존 지지 인덱스만으로 계산되는" 콘텐츠 갭 부류에 속한다는 기존 서술은 정확했지만, "일주 인덱스 나머지 연산 한 줄"이라는 표현은 실제로는 아래처럼 "천간·지지 인덱스 차이 → +10/+11 두 지지"를 구하는 구체적 공식으로 교체한다.

## 계산 공식 — 오픈소스 코드 2종으로 교차 검증
일간 인덱스(갑=0~계=9)를 `s`, 일지 인덱스(자=0~해=11)를 `b`라 하면:

```
diff = (b - s + 12) % 12
공망 지지1 = 지지[(diff + 10) % 12]
공망 지지2 = 지지[(diff + 11) % 12]   // 공망 지지1의 바로 다음 지지
```

이 공식은 `saju`(Dart) 패키지 `core/sinsals.dart`의 실제 구현과 글자 그대로 일치한다(2026-07-18 GitHub 소스 직접 확인, `gracefullight/pkgs` 저장소 `packages/saju-dart/lib/src/core/sinsals.dart`):
```dart
// 공망 (일주 기준)
final sIdx = dayStem.index;
final bIdx = dayBranch.index;
final diff = (bIdx - sIdx + 12) % 12;
final empty1 = Branch.values[(diff + 10) % 12];
final empty2 = Branch.values[(diff + 11) % 12];
```

manseryeok `src/features/void-branches.ts`(2026-07-18 GitHub 소스 직접 확인, `yhj1024/manseryeok` 저장소)는 겉보기엔 다른 방식(60갑자 순번 `dayGanji`을 직접 씀)을 쓰지만 수학적으로 완전히 동치다:
```ts
const dayGanji = ganjiIndexOf(stemIndex, branchIndex); // 60갑자 순번(0~59)
const xunStartBranch = (dayGanji - (dayGanji % 10)) % 12; // 그 순의 첫 간지(갑X)의 지지
return [지지[(xunStartBranch + 10) % 12], 지지[(xunStartBranch + 11) % 12]];
```
`xunStartBranch`(순의 시작 간지가 갖는 지지 인덱스)와 `diff`(saju-dart 방식의 천간·지지 인덱스 차)는 항상 같은 값이 나온다 — 순의 시작 간지는 언제나 천간이 갑(인덱스 0)이므로 그 간지에서는 `diff = 지지 - 0 = 지지`가 되고, 이는 정확히 `xunStartBranch`와 같기 때문이다. **서로 다른 저장소의 두 개발자가 독립적으로 짠 코드가 수학적으로 정확히 동일한 결과를 낸다**는 점에서 [십이운성.md](십이운성.md)의 "독립적 수학적 재계산" 검증과 같은 성격의 교차 확인이다.

## 60갑자 → 공망 대응표 (6개 순 × 각 순의 공망 2지지)
아래 표는 위 공식으로 60갑자 전체(0~59)를 직접 계산해 도출했고, [정해만세력 20강](https://guide.8-codes.com/lecture/elementary/20.html)의 순중공망 표(갑자순=술해, 갑술순=신유, 갑신순=오미, 갑오순=진사, 갑진순=인묘, 갑인순=자축) 및 일반 웹 검색으로 취합한 "60갑자 공망표"(각 순에 속한 10개 일주를 나열하는 blog.daum.net·saju-master.com·sajustudy.com 등 다수 출처)와 전부 일치함을 확인했다(2026-07-18).

| 순(旬) | 소속 60갑자 10개 | 공망 지지 2개 |
|---|---|---|
| 갑자순 | 갑자·을축·병인·정묘·무진·기사·경오·신미·임신·계유 | 술(戌)·해(亥) |
| 갑술순 | 갑술·을해·병자·정축·무인·기묘·경진·신사·임오·계미 | 신(申)·유(酉) |
| 갑신순 | 갑신·을유·병술·정해·무자·기축·경인·신묘·임진·계사 | 오(午)·미(未) |
| 갑오순 | 갑오·을미·병신·정유·무술·기해·경자·신축·임인·계묘 | 진(辰)·사(巳) |
| 갑진순 | 갑진·을사·병오·정미·무신·기유·경술·신해·임자·계축 | 인(寅)·묘(卯) |
| 갑인순 | 갑인·을묘·병진·정사·무오·기미·경신·신유·임술·계해 | 자(子)·축(丑) |

같은 순에 속한 10개 일주는 모두 동일한 공망 2지지를 공유한다 — [신살.md](신살.md)의 12신살 조견표(년지 기준 삼합 그룹으로 12지지 전체를 고정 배치)와 마찬가지로 "그룹 소속 여부만으로 정해지는 고정 배치" 원리라 두 문서의 계산 방식은 개념적으로 모순되지 않는다. 다만 신살은 **년지**(또는 일지) 기준으로 그룹을 나누는 반면, 공망은 **일주 자체(천간+지지 조합)**가 그룹(순) 결정 기준이라는 차이가 있다 — 신살처럼 "년주 따로, 일주 따로 계산해서 합치는" 방식이 아니라 일주 하나로 확정되는 단일 값이라는 점도 명확히 짚어둔다.

`saju`(Dart) 패키지는 별도 파일로 분리하지 않고 신살 계산(`core/sinsals.dart`) 안에서 공망도 함께 산출한다(위 공식, 일간·일지 인덱스 기반) — manseryeok이 `void-branches.ts`로 분리한 것과는 다른 설계. 콘텐츠 갭 7가지 중 난이도 ★☆☆☆☆(가장 쉬움) 평가는 그대로 유지 — 절기 계산이 전혀 필요 없고, 이미 확정된 일간·일지 인덱스만으로 산출되는 순수 조회형 로직이기 때문이다.

## 경쟁 앱들의 노출 카피 — 대부분 "제공 여부"만 표기, 상세 카피는 드묾
- **만세력 천을귀인**(App Store): 기능 목록에 "공망 제공"이라고만 짧게 언급 — 별도의 설명 문구나 UI 톤은 확인되지 않음.
- **포스텔러**: 스토어 설명에 공망 자체는 언급 없이 "12신살과 운성으로 보는 하루 총운"처럼 다른 신살 요소와 뭉뚱그려 소개.
- 즉, 조사한 범위에서는 대형 경쟁 앱들이 공망을 UI 카피 차원에서 정교하게 다루지 않고 "있다/없다" 배지 수준으로만 노출하는 경우가 많다 — 우리 앱이 카피 톤을 신경 쓴다면 차별화 여지가 있어 보이나, 이는 관찰일 뿐 구현 판단은 사람 결정 대기.

## 명리학 콘텐츠에서 반복되는 톤 패턴 — "부정적 인식 인정 → 극단적 해석 경계 → 양면 재해석"
여러 명리 칼럼(브런치, 사주 블로그)에서 공통적으로 나타나는 3단 구조:
1. **부정적 인식을 먼저 인정**: "사주에 공망이 있으면 좋지 않은 의미로 느껴진다"는 통념을 부정하지 않고 그대로 언급.
2. **극단적 해석을 경계**: "그렇게 극단적으로 생각할 필요는 없다"고 톤을 낮춘다.
3. **양면적 재해석 제시**: "비어 있어서 힘을 못 쓴다"는 원의미를 "비어 있기에 채울 수 있고, 없기에 덜 집착할 수 있는 자유"로 재프레이밍. 공망을 "결핍이 아니라 유연성의 상징"으로 보는 표현도 발견됨.

이 패턴은 앱 카피를 작성할 때 참고할 만한 톤 가이드다(사람 결정 시): 공망이 "있다"고 무조건 불안하게 표현하기보다, 통념(약함/결핍) → 균형 잡힌 재해석(자유/유연성)의 2단 구성이 명리 콘텐츠 업계에서 널리 쓰이는 접근이다.

### "공망 해소법" 콘텐츠 앵글 (일부 블로그 사례)
1. 기대 낮추기 — 공망 위치 영역에서 과한 기대를 버리기
2. 운세로 보완 — 공망된 지지를 보완하는 시기/행운 활용
3. 내면 채우기 — 명상·예술활동·상담 등으로 "내적 허기" 채우기
4. 재해석 — 결핍이 아니라 유연성의 상징으로 보기

### 궁합에서 공망이 겹칠 때
같은 공망 지지를 공유하는 두 사람을 "감정선이 비슷하고 서로의 빈 부분을 이해하는 관계", "같은 결핍을 공유하며 동질감으로 신뢰를 형성"하는 관계로 해석하는 블로그 사례가 있음 — 궁합 콘텐츠(`docs/research/궁합/`)에 공망을 연동할 경우 참고할 만한 각도이나, 이는 궁합 기능과 공망 기능이 모두 구현된 이후에나 유효한 확장 아이디어.
