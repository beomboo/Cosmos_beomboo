# 택일(擇日) — 콘텐츠 갭이 아니라 별도 제품 형태

_최종 확인: 2026-07-13_

**현황**: 미구현. "사주냥" 마케팅 자료에서 신살과 함께 발견. 구현 여부는 사람 결정 대기.

## 정의
"좋은 날을 고른다"는 뜻으로, 결혼·이사·출산(제왕절개 등) 같은 미래 일정에 대해 음양오행·사주·절기·풍수를 종합해 길일을 찾아주는 서비스.

## 왜 다른 콘텐츠 갭(십신·용신·대운/세운·공망·십이운성·신살)과 근본적으로 다른가
그 6가지는 전부 **"이미 정해진 한 사람의 생년월일시(BirthInfo) → 그 명식을 해석"**하는 동일한 입출력 모델을 공유한다. 택일은 **"목적(결혼/이사/출산)+탐색 대상 기간 → 그 기간 안에서 길일 후보를 스캔해 달력으로 보여주기"**로 입력·출력 모델 자체가 다르다(결혼 택일은 두 사람의 명식을 동시에 고려하기도 함). 기존 계산·콘텐츠 자산을 재사용하기 어렵고 새로운 화면 흐름(날짜 범위 선택, 달력 UI 결과 표시)이 필요해 "심층 분석처럼 확장 화면 하나 추가" 패턴과는 규모가 다르다.

**결론**: 콘텐츠 갭 종합표에 억지로 끼워 넣기보다, "이 앱의 핵심 정체성(개인 명식 해석)과는 다른 종류의 잠재적 신규 기능"으로 별도 분류한다.
