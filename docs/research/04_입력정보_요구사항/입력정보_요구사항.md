# 입력 정보 요구사항 (온보딩 설계 참고)

## 필수 입력값
| 항목 | 비고 |
|---|---|
| 생년월일 | 양력/음력 선택 토글 필요 |
| 태어난 시간 | 24시간 기준, 분까지 받으면 정확도 상승 |
| 성별 | 대운(순행/역행) 계산에 필요 |

## 선택/고급 입력값
| 항목 | 비고 |
|---|---|
| 출생지(도시) | 진태양시 보정(경도)에 사용 — 없으면 표준시로 근사 계산 |
| 윤달 여부 | 음력 입력 시 윤달인지 표시 필요 |

## "태어난 시간을 모를 때" 처리 — 앱별 접근 차이
- **엄격 요구형**: 시간 입력을 필수로 강제하되 "모름" 옵션은 두고, 시주 없이 근사 결과 제공 (신운세 등)
- **유연 허용형**: 시간 없이도 진행 가능, 대신 "시주가 빠진 사주는 해석의 일부만 가능"이라고 안내
- **UX 권장안**: "모름" 선택 시 시주만 제외한 정통 사주(년/월/일주 3개)를 보여주고, "태어난 시간을 알면 더 정확한 결과를 볼 수 있어요" 식으로 재입력을 유도하는 넛지 문구 배치

## 자시(子時) 관련 안내 필요
23:00~01:00 출생자는 일주가 전날/당일 중 어느 쪽으로 계산되는지 앱마다 다름. 이 구간 입력 시 "정확한 결과를 위해 병원 기록상 시간을 확인해보세요" 같은 툴팁 고려.

### 자시 처리 3가지 관법 실측 (2026-07-06, manseryeok 소스코드 확인) — 우리 앱의 현재 선택 포함
07_오픈소스_계산로직에서 manseryeok의 절기 계산을 분석하다가(`src/pillars.ts`), 이 라이브러리가 23:00~23:59(자시) 출생을 처리하는 방식을 `dayBoundary` 옵션으로 3가지 모두 명시적으로 지원하는 것을 확인했다 — 명리학계에서도 실제로 유파마다 갈리는 지점이라는 걸 코드 수준에서도 보여주는 사례라 정리해둔다.

| 관법 | 일주(日柱) | 시주(時柱) 천간 기준 일간 | 비고 |
|---|---|---|---|
| `midnight` | 당일 그대로 | 당일 그대로 | 자시 롤오버 없음(가장 단순) |
| `splitJasi`(조자시/야자시 분리) | 당일 그대로 | **다음날**로 롤오버 | 전통 명리학에서 가장 흔히 언급되는 관법 |
| `jasi`(자시 전체 롤오버) | **다음날**로 롤오버 | 다음날로 롤오버 | 일주까지 통째로 다음날 취급 |

**우리 앱(`project/lib/core/saju/four_pillars.dart`)의 현재 동작**: 위 표의 `midnight` 관법과 정확히 일치한다 — `calculateFourPillars`가 `dayStemIndex`를 오직 `birthDate`(달력 날짜)만으로 계산하고(62~66행), 이 값을 시주 천간 계산(77행)에도 그대로 재사용해 자시 롤오버 자체가 없다. 이 선택은 코드 어디에도 명시적으로 문서화돼 있지 않았음(절기 근사 한계는 상단 doc-comment에 있지만 자시 관법은 언급 없음) — 이번에 확인한 내용으로 처음 명시.

**시사점**: `midnight`은 세 관법 중 가장 단순하지만 전통 명리학에서 더 흔히 쓰이는 `splitJasi`와는 23:00~23:59 출생자의 시주(정확히는 시주의 천간)가 달라질 수 있다. 이는 절기 근사(며칠 단위 오차)와는 별개의 정확도 이슈이므로, 향후 "정밀 계산" 결정을 내릴 때 절기 문제와 함께 이 자시 관법 선택도 같이 검토 대상에 넣을 것을 제안한다. (구현 여부는 여전히 사람 결정 대기 — PROJECT_ROUTER.md의 "사주 계산 로직" 행 참고)

## 온보딩 흐름 참고 (경쟁 앱 공통 패턴)
1. 생년월일시 입력 화면 (달력 UI + 시간 스크롤 피커)
2. 양력/음력 토글
3. 성별 선택
4. (선택) 출생지 입력 — 스킵 가능하게
5. 로딩 애니메이션 (사주 계산 중 — 이 구간에 캐릭터/일러스트 연출 넣기 좋은 지점)
6. 결과 화면 진입

## UX 상 주의점
- 입력 필드가 많으면 이탈률 상승 → 필수값 최소화 + 선택값은 "나중에 추가" 유도
- 시간 모름 케이스가 실제로 매우 흔함 (병원 기록 없는 경우 다수) → 이 케이스의 결과 품질/설명이 앱 신뢰도에 직결
