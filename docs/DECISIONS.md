# 사람 결정 대기 목록

이 문서는 `CLAUDE.md` 규칙 6번의 게이트 항목과 그 밖의 사람 결정 대기 사안을 한곳에 모은
승인 창구입니다. 대화 세션이 끊겨도(특히 오버나이트 자동화 루프처럼 세션 기억이 없는
실행에서도) 여기 적힌 답변만으로 다음 사이클이 이어서 처리할 수 있도록 설계했습니다.

## 사용법

- 각 항목의 **상태** 줄이 `⬜ 대기` / `✅ 승인` / `⛔ 반려` / `🔧 처리 완료` 넷 중 하나입니다.
- **답변해주실 부분**: `⬜ 대기` 항목의 "사용자 답변" 아래 빈 칸에 의견을 적고, 상태를
  `✅ 승인`(제안대로 진행) 또는 `⛔ 반려`(하지 않음/다른 방향)로 바꿔주시면 됩니다. 제안과
  다른 방향을 원하시면 "사용자 답변"에 그 방향을 적어주세요 — 오버나이트 루프는 상태가
  `✅ 승인`이면 "사용자 답변"에 적힌 내용(비어 있으면 "추천" 내용)을 그대로 승인된 요구사항
  삼아 `CLAUDE.md` 규칙 6번 절차(saju-planner → saju-developer → saju-tester)로 넘깁니다.
- 처리(구현 또는 명시적으로 하지 않기로 확정)가 끝나면 담당 세션이 상태를 `🔧 처리 완료`로
  바꾸고 결과를 한 줄 남깁니다 — 이 문서는 완료 이력까지 계속 쌓아두는 결정 로그이므로,
  `PROJECT_ROUTER.md`와 달리 완료된 항목도 지우지 않고 상태만 바꿔 남겨둡니다.
- 오버나이트 루프(`COSMOS_LOOP_JOB.md`)는 매 사이클 시작 시 이 문서에서 `✅ 승인` 상태인
  항목이 있는지 먼저 확인합니다.

## D. AI 해석 연동

**상태**: ⬜ 대기
**질문**: 지금은 모든 해석 문구가 정적 텍스트(예: `ohaeng_readings.dart`)입니다. LLM을 연동해 명식별로 동적 해석 문구를 생성하는 방향을 도입할까요?
**추천**: 판단 보류 — 규칙 7번("계산 자체는 고정, 풀이는 차별화 가능")과 충돌하지 않도록 "계산 결과는 그대로 두고 그걸 설명하는 문장만 AI가 생성"하는 경계를 명확히 해야 하는 게이트 항목. 도입한다면 프롬프트에 원문 계산값을 고정 입력으로 강제하고 AI가 계산 자체를 재추론하지 않게 설계하는 것을 전제로 제안.
**사용자 답변**:
>
